Đảm bảo được 3 yêu cầu:
1) UI giống với thiết kế.
2) Thu/phóng sidebar thông qua nút ở trên
3) Khi click vào item ở sidebar thì bỏ active item cũ đồng thời active item vừa được click

Hướng dẫn:

Viết css cho trạng thái mặc định (trạng thái phóng to) của sidebar - tạm gọi là class 'sidebar'
Viết css cho trạng thái thu nhỏ của sidebar - tạm gọi là class 'mini-sized'. Đảm bảo được việc css của class 'mini-sized' sẽ ghi đè được css của class 'sidebar'
=> sidebar sẽ luôn mặc định có class 'sidebar', khi muốn thu nhỏ ta thêm class 'mini-sized', làm tương tự khu muốn phóng to ra
Tạo function để thực hiện việc thêm/xóa class bằng các methods của classList (nên sử dụng toggle). Gán function này vào nút trên header

Viết css cho 2 trạng thái thường và active.
Thêm class 'active' vào phần tử cần được active
logic là khi click vào 1 item ở sidebar ta làm 2 việc:
B1: xóa toàn bộ class active hiện đang có - mục đích để tránh việc có nhiều hơn 1 phần tử được active
=> cần lấy danh sách CÁC phần tử con của sidebar VÀ đang được active. Tiến hành lặp qua từng phần tử và xóa class active.
B2: Thêm class 'active' vào phần tử được click, do sau bước 1 ta đã xóa hết phần tử active nên giờ ta chỉ cần thêm mới phần tử được tác động là được
=> xác định phần tử được click là phần tử nào. Thêm mới class active vào phần tử.
